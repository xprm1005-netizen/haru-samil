# CLAUDE.md

이 프로젝트는 "하루삼일" 서비스의 초기 제품 기획 및 프론트엔드 MVP를 만들기 위한 작업 공간입니다.

## Product Principle

하루삼일은 일반적인 할 일 앱이 아닙니다. 사용자가 24시간을 하나의 하루로 느끼지 않고, 8시간마다 새 하루를 시작하는 감각을 주는 앱입니다.

가장 중요한 사용자 감정:

> 아, 오늘 아직 안 끝났구나.

## Build Priorities

1. 첫 화면에서 "오늘의 몇 일차인지" 즉시 보여준다.
2. 사용자가 입력해야 하는 것은 각 일차 목표 1개뿐이다.
3. 실패라는 표현보다 "놓쳤다", "버텼다", "해냈다"를 쓴다.
4. 재시작 버튼은 핵심 기능이다.
5. 기능보다 문장과 시간 감각이 중요하다.

## UX Copy Rules

- "오전/오후/저녁"보다 "1일차/2일차/3일차"를 우선 사용한다.
- 사용자를 탓하는 문장을 쓰지 않는다.
- 자기계발 앱처럼 과하게 뜨겁거나 훈계하지 않는다.
- 짧고 단단한 문장을 쓴다.

Good:

- 오늘의 2일차가 시작됐습니다.
- 이번 하루에 하나만 해내세요.
- 첫 번째 하루는 끝났고, 새 하루가 열렸어요.
- 놓쳤어도 괜찮습니다. 다음 하루가 있습니다.

Avoid:

- 실패했습니다.
- 의지가 부족했어요.
- 목표를 반드시 달성하세요.
- 성공하는 사람은 다릅니다.

## Frontend Direction

- 모바일 우선
- 알람 앱처럼 단순한 구조
- 큰 시간 숫자
- 현재 일차 상태를 강하게 표시
- 카드 남발 금지
- 중첩 카드 금지
- 일차별 색상은 구분용으로만 사용

## Suggested Stack

초기 구현은 웹 MVP 기준으로 시작해도 좋습니다.

- React 또는 Next.js
- TypeScript
- Tailwind CSS
- Zustand 또는 React state
- LocalStorage 기반 저장

네이티브 앱으로 갈 경우:

- Expo React Native
- AsyncStorage
- Local notification
